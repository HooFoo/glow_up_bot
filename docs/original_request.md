## Концепция проекта

Телеграм-бот на стеке **PHP + MySQL**.

Проект: **Prime/Glow Ассистент** — персональный AI-консультант по питанию, косметике и коучингу.

---

## Пользовательский путь

Пользователь попадает в бота и проходит **Квиз**, на основе которого заполняется его профиль. По результатам квиза пользователю присваивается его **«типаж»** — с описанием и картинкой.

**Подарок при регистрации:** доступ к боту на **один день** бесплатно. После этого бот закрывается **пейволом**.

---

## Режимы бота

В дальнейшем в боте есть три режима:

1. **Консультант по питанию**
2. **Консультант по косметике**
3. **Коуч**

Во всех режимах бот использует:
- Системный промпт (из файла `.md`)
- Профиль пользователя
- Резюме предыдущих разговоров
- Последние 5 сообщений как контекст

Для ответов используется **LLM (ChatGPT)** через OpenAI API.

---

## Механика работы бота

- **Каждые 5 сообщений** — сжатие переписки в резюме
- **Каждые 10 сообщений** — обновление профиля пользователя (добавление фактов из последних 10 сообщений)

---

## Мультимодальность

- **Голосовые сообщения** — преобразуются в текст (Speech-to-Text через OpenAI Whisper)
- **Фото** — обрабатываются с учётом текстового комментария под ним (Vision API)
- **Ответ Telegram:** Бот должен отвечать HTTP 200 сразу после получения запроса (используя `fastcgi_finish_request`), чтобы Telegram не ждал генерации.
- **Индикация (ChatEvents):** Использование `sendChatAction` («typing») во время обработки и генерации ответа.

---

## Пейвол и подписка

- Без подписки: доступ на **первые 2 дня** с момента прохождения квиза
- **Подписка:** на один месяц
- Оплата: **Telegram Stars**

---

## Административная панель (веб)

### Список пользователей
- Сортировка: сначала с активной подпиской, затем без подписки
- Внутри групп — сортировка по дате регистрации
- Отображение: username пользователя
- Пагинация

### Страница пользователя
- Данные для входа (Telegram ID, username, имя)
- Количество сообщений
- Статус подписки и дата окончания
- Профиль пользователя
- Резюме переписки

### Страница переписки
- Все сообщения от пользователя и от бота
- Формат: визуальная переписка (чат)
- Медиафайлы не сохраняются

### Dashboard
- Общее количество пользователей
- График новых сессий по дням (за 10 дней)
- График покупки подписок по дням (за 10 дней)
- Количество активных подписок
